# Round26 前端业务化硬化报告

## 本轮目标
把前台从技术控制台收口成客服可直接使用的业务界面，并保持 Round25 的后端硬化成果。

## 已完成
- 将 React 前端主要页面统一改为中文业务文案：首页总览、工单处理、通知公告、渠道设置、系统状态。
- 去除前台显性技术品牌词，避免在客服页面暴露 OpenClaw / NexusDesk 等字样。
- 新增展示文案清洗：运行提醒、上线检查等来自后端的文本会在前端替换技术词。
- 重做前端样式层：信息层级更清晰、色彩更稳、卡片与表单更适合客服日常操作。
- 保留后台真实 API 与字段，不影响接口协议和 Round25 的 AI intake 500 修复。
- 兼容 legacy frontend：即使未先构建 webapp，旧前端入口也已做基础去术语处理。

## 风险与说明
- 后端内部字段名仍保留 openclaw_*，这是协议层与数据层命名，不在前台直接展示。
- legacy frontend 仅做基础去术语与文案收口，主推荐入口仍是 webapp 构建产物。

## 验证
- `webapp/npm ci && npm run build` 通过。
- 新构建产物会输出到 `frontend_dist/`，可供 FastAPI 直接挂载。

## 建议下一轮
- 增加客服视角的页面级 e2e 冒烟测试。
- 继续压缩 legacy frontend，逐步退役，仅保留 React 前端。
- 将常用状态、优先级、告警文案完全本地化到统一字典。
