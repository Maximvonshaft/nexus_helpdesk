# Design Review Checklist

- [ ] Email channel is not visible as sendable when capability says not ready.
- [ ] Disabled reason is visible and actionable.
- [ ] Agent can see From/To/Subject before sending.
- [ ] Send result is visible in timeline.
- [ ] Bounce/complaint states are visually distinct.
- [ ] No secret/provider internals shown to agents.
- [ ] UI handles loading/error/empty states.
- [ ] No fake button or mock-only UI.
- [ ] Mobile/narrow layout remains usable if existing ticket page supports it.
- [ ] Copy is operator-friendly.
