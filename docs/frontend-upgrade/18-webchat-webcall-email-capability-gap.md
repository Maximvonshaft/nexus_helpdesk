## Explicit product decisions

- Top-level `/webcall` is the operator workbench entry; `/webcall/$voice_session_id` remains the customer voice room and `/webchat-voice` remains a legacy internal deep link.
- `/email` draft save is implemented against the existing backend endpoint; it is not marked as a fake frontend-only save.
- Email attachment send support is implemented for external ticket attachments, including inline upload from the Email composer through `/api/tickets/{ticket_id}/attachments`.
- Email provider retry controls reuse the existing admin outbound requeue API and stay `runtime.manage` gated inside `/email`; this avoids giving ordinary outbound senders runtime recovery permissions.
- Email outbound mailbox identity is implemented for ticket-bound draft/send rows and SMTP headers. Inbound provider-event ingest, delivery receipt ingest, mailbox queue projection, and continuous IMAP polling are implemented as ticket-scoped backend paths with explicit RBAC and audit/timeline evidence.
- Email queue selection is now backend-owned through `/api/email/queue`. The projection accepts real inbound mailbox rows, real Email outbound rows, and explicit Email ticket markers; it does not use loose browser-side substring filtering.
