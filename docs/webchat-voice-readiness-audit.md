# Voice readiness authority

## Code-complete scope

- one LiveKit media plane
- browser and SIP sessions
- explicit AI Agent dispatch
- canonical human handoff ownership
- independent voice capacity and wrap-up
- signed, idempotent webhook inbox
- SIP transfer and outbound participant commands
- channel configuration UI
- permanent retired-media residue gate

## External activation evidence still required

A deployment is not live-telephone-ready until an authorized operator supplies and verifies LiveKit credentials, DID/Trunk configuration, provider transfer policy, DNS/TLS, firewall/RTP reachability, test-call evidence and country-specific recording policy. These are intentionally fail-closed.
