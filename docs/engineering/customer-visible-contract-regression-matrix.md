# Customer Visible Contract Daily Smoke and Regression Matrix

## Daily smoke scope

| Area | Scenario | Expected result | Command / owner |
|---|---|---|---|
| WebChat | Greeting message: `你好` | AI reply is generated by runtime, not hardcoded text. Public message has AI-origin outbound record with trace/signature/contract fields. | `python scripts/smoke_customer_visible_contracts.py --base-url <url> --channel webchat --message "你好" --expect-ai-reply` |
| WebChat | Human request: `转人工` / `人工客服` | Conversation enters `human_review_required` or equivalent handoff state. No hardcoded handoff ack is created. | WebChat smoke + DB audit |
| Handoff | Runtime requests human review | Handoff state/event exists. Any customer-visible handoff notice must be AI origin + v3 `handoff_notice`; otherwise status/event only. | DB audit + handoff queue review |
| Outbound | Signed body mutation simulation | Outbound is dead with `failure_code=runtime_signed_body_mutation`. | `pytest backend/tests/test_ai_customer_visible_contracts.py -q` |
| Outbound | Originless outbound simulation | Outbound is dead with `failure_code=missing_customer_visible_origin_contract`. | `pytest backend/tests/test_ai_customer_visible_contracts.py -q` |
| Human reply | Agent sends reply after takeover | `origin=human_agent`, `created_by` exists, `last_human_update` may update, runtime trace is not required. | Staging/manual smoke + DB audit |
| WhatsApp | AI reply queued to native adapter | Outbound origin remains `provider_runtime`, contract fields exist, sidecar is not changed. | Existing WhatsApp adapter tests |
| Knowledge | Country-specific retrieval | Country-specific knowledge beats GLOBAL; no cross-country policy leakage. | `pytest backend/tests/test_knowledge_runtime_seed_contract.py -q` |
| Knowledge | Quality audit | GLOBAL fallback, no-hit, high-risk no-source, and handoff-after-hit are visible. | `python scripts/audit_knowledge_runtime_quality.py --hours 24 --format json` |
