# Nexus Native WhatsApp Candidate Smoke Runbook

Scope: validate the Nexus-native WhatsApp sidecar path beside production without
changing public nginx routing or replacing the current production app.

This runbook intentionally avoids secrets. Load all tokens, connector keys,
database URLs, and image tags from the server secret store or the release
environment.

## Preconditions

- Candidate image has already been built and pushed.
- `deploy/.env.candidate` exists on the server and is not committed to Git.
- `WHATSAPP_NATIVE_ENABLED=true`.
- `WHATSAPP_DISPATCH_MODE=native_sidecar`.
- `WHATSAPP_SIDECAR_URL=http://whatsapp-sidecar-candidate:18793`.
- `CANDIDATE_NEXUS_BACKEND_URL=http://app-candidate:8080`.
- Speedaf write flags remain disabled until a separate controlled-action UAT:
  - `SPEEDAF_WORK_ORDER_CREATE_ENABLED=false`
  - `SPEEDAF_UPDATE_ADDRESS_ENABLED=false`
  - `SPEEDAF_CANCEL_ENABLED=false`
  - `SPEEDAF_VOICE_CALLBACK_ENABLED=false`

## 1. Validate Compose Before Start

Run from the repository root on the server:

```bash
COMPOSE_PROJECT_NAME=nexusdesk_candidate docker compose \
  -f deploy/docker-compose.candidate.yml \
  --env-file deploy/.env.candidate \
  config --quiet
```

Then inspect the rendered graph:

```bash
COMPOSE_PROJECT_NAME=nexusdesk_candidate docker compose \
  -f deploy/docker-compose.candidate.yml \
  --env-file deploy/.env.candidate \
  config | grep -E 'app-candidate|worker-outbound-candidate|whatsapp-sidecar-candidate|app-candidate:8080'
```

Fail the rollout if the rendered config contains:

```text
NEXUS_BACKEND_URL: http://app:8080
```

## 2. Start Candidate Only

Do not change nginx yet.

```bash
COMPOSE_PROJECT_NAME=nexusdesk_candidate docker compose \
  -f deploy/docker-compose.candidate.yml \
  --env-file deploy/.env.candidate \
  up -d app-candidate worker-outbound-candidate whatsapp-sidecar-candidate
```

Check container health:

```bash
COMPOSE_PROJECT_NAME=nexusdesk_candidate docker compose \
  -f deploy/docker-compose.candidate.yml \
  --env-file deploy/.env.candidate \
  ps
```

## 3. Candidate App Smoke

```bash
BASE_URL=http://127.0.0.1:18082 \
REQUIRE_RELEASE_METADATA_COMPLETE=true \
bash scripts/smoke/production_candidate_smoke.sh
```

This validates:

- `/healthz`
- `/readyz`
- `/webchat/demo/`
- `/webchat/voice-entry.js`
- WebChat CORS allow/block behavior
- release metadata identity when expected values are supplied

## 4. Sidecar Health And QR Smoke

Use the same token configured in `deploy/.env.candidate`; do not print it.

```bash
WA_SIDECAR_BASE_URL=http://127.0.0.1:18795 \
WA_SIDECAR_INTERNAL_TOKEN="$WA_SIDECAR_INTERNAL_TOKEN" \
WA_ACCOUNT_ID=wa-main \
WA_SIDECAR_EXPECT_MODE=baileys \
WA_SIDECAR_SMOKE_START_LOGIN=true \
WA_SIDECAR_EXPECT_QR_OR_CONNECTED=true \
bash scripts/smoke/whatsapp_sidecar_candidate_smoke.sh
```

Expected result:

- sidecar `/healthz` returns `status=ok`
- sidecar `/readyz` returns `status=ready` and `mode=baileys`
- account status is readable through the bearer token
- account is either `qr_pending` with QR data or already `connected`

## 5. Bind WhatsApp

Open the Nexus admin WhatsApp account page against the candidate backend or use
the QR evidence generated by the smoke script. Scan the QR with the intended
WhatsApp device.

After scanning, verify the connected state:

```bash
WA_SIDECAR_BASE_URL=http://127.0.0.1:18795 \
WA_SIDECAR_INTERNAL_TOKEN="$WA_SIDECAR_INTERNAL_TOKEN" \
WA_ACCOUNT_ID=wa-main \
WA_SIDECAR_EXPECT_MODE=baileys \
WA_SIDECAR_EXPECT_ACCOUNT_STATUS=connected \
bash scripts/smoke/whatsapp_sidecar_candidate_smoke.sh
```

## 6. Optional Live Send Smoke

Run this only with a controlled test recipient. This sends a real WhatsApp
message.

```bash
WA_SIDECAR_BASE_URL=http://127.0.0.1:18795 \
WA_SIDECAR_INTERNAL_TOKEN="$WA_SIDECAR_INTERNAL_TOKEN" \
WA_ACCOUNT_ID=wa-main \
WA_SIDECAR_EXPECT_MODE=baileys \
WA_SIDECAR_SMOKE_SEND=true \
WA_SIDECAR_ALLOW_LIVE_SEND=true \
WA_SIDECAR_SMOKE_SEND_TARGET='<test-phone-or-jid>' \
WA_SIDECAR_SMOKE_SEND_BODY='NexusDesk candidate WhatsApp smoke' \
bash scripts/smoke/whatsapp_sidecar_candidate_smoke.sh
```

## 7. Inbound Smoke

From the controlled test WhatsApp account, send a message to the bound number.
Verify in the candidate backend:

- WhatsApp native inbound callback succeeds.
- A WebChat projection or ticket is created as expected.
- AI fast reply remains routed through Nexus provider runtime.
- No retired vendor runtime bridge or retired vendor runtime gateway is required for this path.

Use application logs and tool-governance logs to confirm the path:

```bash
COMPOSE_PROJECT_NAME=nexusdesk_candidate docker compose \
  -f deploy/docker-compose.candidate.yml \
  --env-file deploy/.env.candidate \
  logs --tail=200 app-candidate worker-outbound-candidate whatsapp-sidecar-candidate
```

## 8. Stop Candidate / Rollback

Because nginx was not changed, rollback means stopping the candidate services:

```bash
COMPOSE_PROJECT_NAME=nexusdesk_candidate docker compose \
  -f deploy/docker-compose.candidate.yml \
  --env-file deploy/.env.candidate \
  down
```

Do not delete the WhatsApp session directory until the test result has been
reviewed. If a clean QR rebind is required, remove only the candidate session
directory configured by `CANDIDATE_WHATSAPP_SESSION_ROOT`.

## Promotion Gate

Only consider public routing after all are true:

- GitHub `candidate-whatsapp-native-gate` passes.
- Candidate compose config renders without the legacy `app:8080` callback.
- Candidate app smoke passes.
- Sidecar QR/connected smoke passes.
- Controlled inbound message appears in Nexus candidate data.
- Optional live send smoke passes, if outbound WhatsApp is in scope.
- Rollback command has been tested or is operationally ready.
